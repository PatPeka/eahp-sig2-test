Candidate use cases

Initial interoperability scenarios

UC-01

Patient-specific dispense order from PIS to preparation system

Define the minimum information needed to transmit a validated patient-specific dispense order from the pharmacy information system.

UC-02

Unit-dose labeling and traceability

Clarify label data, identifiers, batch references, and traceability requirements for prepared unit doses.

UC-03

Order cancellation or modification after production planning

Describe how late clinical or operational changes should be communicated after preparation work has been scheduled.

UC-04

Preparation confirmation and status feedback

Define the status messages returned to the PIS after acceptance, rejection, preparation, or completion.

Process flow

Baseline dispense-order pathway

  1. Prescription validated
  2. Dispense order created
  3. Order transmitted
  4. Preparation system accepts/rejects
  5. Unit dose prepared
  6. Confirmation returned

Proposed decisions

Items to confirm during Call #1

D-001

Use case documentation will follow a standard SOP template.

D-002

Each selected use case will include a flow diagram first, then BPMN where appropriate.

D-003

Each use case will maintain separate pharmacist and vendor validation questions.

Validation questions

Keep pharmacist and vendor input separate

For pharmacists

  • Which clinical checks must happen before an order is sent?
  • Which label data are mandatory for safe administration?
  • Which status changes must be visible in the PIS?

For vendors

  • Which message formats and identifiers are currently supported?
  • How are rejected, cancelled, or modified orders handled?
  • Which confirmation events can be returned automatically?
